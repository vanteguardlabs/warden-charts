  curl localhost:8084/readyz   # hil

  curl localhost:8086/readyz   # identity
  curl localhost:8087/readyz   # deep-review

Proxy serves /mcp + /metrics on its mTLS port (8443) only. /health
and /readyz are *also* on a non-mTLS health listener (8080 by default,
container-port only — not published by the Service). Kubelet probes
target 8080. To exercise the mTLS path locally, point a client with
valid certs at https://localhost:8443/ per warden-proxy/README.md.

{{- if not .Values.tlsBundle.secretName }}

WARNING: tlsBundle.secretName is empty.

The proxy panics at boot without /certs mounted (see warden-proxy
recurring-gotchas in CLAUDE.md). Generate a cert bundle with
`warden-proxy/scripts/gen_certs.sh --env prod` then:

  kubectl -n {{ .Release.Namespace }} create secret generic warden-certs \
    --from-file=warden-proxy/certs/

  helm upgrade {{ .Release.Name }} . --reuse-values \
    --set tlsBundle.secretName=warden-certs
{{- end }}
